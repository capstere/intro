Ja — de där är mest accessibility/App Checker-fixar, inte funktionsfel.

Power Apps flaggar Missing accessible label när ett interaktivt kontrollobjekt saknar text i AccessibleLabel. Den flaggar Missing tab stop när kontrollen inte deltar i tangentbordsnavigering, styrt av TabIndex för classic controls. Den flaggar Focus isn’t showing när fokusramen inte syns, typiskt när FocusedBorderThickness är 0. Microsoft rekommenderar också att man normalt använder TabIndex = 0 eller -1 för enkelhet.  ￼

Gör exakt detta

1) txtSearchPN

Sätt:

AccessibleLabel = "Search PN, barcode, or material name"
TabIndex = 0
FocusedBorderThickness = 2

2) galMaterials

Sätt:

AccessibleLabel = "Search results list of materials"
TabIndex = 0
FocusedBorderThickness = 2

Det gör galleryn fokusbar för tangentbordsanvändare, vilket också är viktigt för scrollning i galleries utan tydligt interaktivt innehåll.  ￼

3) galLots

Sätt:

AccessibleLabel = "List of lots for selected material"
TabIndex = 0
FocusedBorderThickness = 2

4) txtLotNumber

Sätt:

AccessibleLabel = "Lot number"
TabIndex = 0
FocusedBorderThickness = 2

5) txtDeleteReason

Sätt:

AccessibleLabel = "Reason for deleting lot"
TabIndex = 0
FocusedBorderThickness = 2

6) dpExpirationDate

Sätt:

AccessibleLabel = "Expiration date"
TabIndex = 0
FocusedBorderThickness = 2

7) txtQty

Sätt:

AccessibleLabel = "Quantity"
TabIndex = 0
FocusedBorderThickness = 2

Var felet låg

Den här raden i din gamla formel var ett bra exempel:

LookUp(CM_Lots; LotKey = ... )

Den ska vara:

LookUp(CM_Lots, LotKey = ... )

Alltså komma i LookUp, inte semikolon.

Om LotKey fortfarande blir rött efter detta

Testa i en tillfällig label:

First(CM_Lots).LotKey

Om den också är röd måste CM_Lots refreshas igen eller tas bort/läggas till på nytt.
