

Ändrat på Logiken i färger för kalendrar med Leading och lagging.

A. Leading-kalendrar — helger blå när ingen status finns
LEADING:

EHS_Leading_CalendarColor
EHS_Leading_CalendarColor =
VAR _status = [EHS_Leading_DayStatus]
VAR _isWeekend = SELECTEDVALUE(Dim_Date[IsWeekend])
RETURN
    SWITCH(
        TRUE(),
        NOT(ISBLANK(_status)) && _status = "Green", 1,
        NOT(ISBLANK(_status)) && _status = "Red", -1,
        _isWeekend = TRUE(), -0.5,
        0
    )

D_Leading_CalendarColor
Använt samma mönster senare när D Leading är modellmässigt klar:

Vad detta gör:
vardagar utan data = vita
lör/sön utan data = ljusblå
riktiga statusar vinner alltid först

LAGGING:
B. Lagging-kalendrar — data vinner, blått bara för prev-month-weekend

Detta är samma mönster som jag fungerar för D Lagging.

EHS_Lagging_CalendarColor
EHS_Lagging_CalendarColor =
VAR _status = [EHS_Lagging_DayStatus]
VAR _isPrevMonth = SELECTEDVALUE(Dim_Date[IsPreviousCalendarMonth])
VAR _isWeekend = SELECTEDVALUE(Dim_Date[IsWeekend])
RETURN
    SWITCH(
        TRUE(),
        NOT(ISBLANK(_status)) && _status = "Green", 1,
        NOT(ISBLANK(_status)) && _status = "Red", -1,
        _isPrevMonth = TRUE() && _isWeekend = TRUE(), -0.5,
        0
    )

Q_Lagging_CalendarColor
Q_Lagging_CalendarColor =
VAR _status = [Q_Lagging_LFI_DayStatus]
VAR _isPrevMonth = SELECTEDVALUE(Dim_Date[IsPreviousCalendarMonth])
VAR _isWeekend = SELECTEDVALUE(Dim_Date[IsWeekend])
RETURN
    SWITCH(
        TRUE(),
        NOT(ISBLANK(_status)) && _status = "Green", 1,
        NOT(ISBLANK(_status)) && _status = "Red", -1,
        _isPrevMonth = TRUE() && _isWeekend = TRUE(), -0.5,
        0
    )

Vad detta gör:
fred, lör, sön kan bli grön/röd om lagging-data matas in på måndag
previous-month lör/sön kan fortfarande vara ljusblå om de bara är layoutöverhäng
du får samma beteende i EHS Lagging och Q Lagging som i D Lagging
