Ja — nu ser jag vad som händer.

Problemet är nästan säkert inte logiken längre. Det är syntaxen i Studio.

Du sitter sannolikt i en miljö där Power Apps vill ha semikolon i formelfältet, även om appen i övrigt är på engelska.
Det är därför du får röda streck över hela record-blocket i både Patch(...)-anropen.

Det här syns tydligt i dina bilder:
	•	hela { ... }-recorden blir röd
	•	varje rad i recorden blir röd
	•	samma mönster i både btnConfirmDelete och btnSaveLot

Det är klassiskt separator-problem.

Gör så här nu

Regel

I Studio-formelfältet ska du använda:
	•	; mellan funktionsargument
	•	; mellan fält i { ... }

Inte kommatecken.

1) Byt btnConfirmDelete.OnSelect till exakt detta

If(
    IsBlank(varSelectedLot);
    Notify("Select a lot first."; NotificationType.Error);
    IfError(
        Patch(
            CM_Lots;
            varSelectedLot;
            {
                IsActive: false;
                ChangedAt: Now();
                ChangedByName: User().FullName;
                ChangedByEmail: User().Email
            }
        );
        Patch(
            CM_Transactions;
            Defaults(CM_Transactions);
            {
                Title: varCurrentMaterial.PN;
                PN: varCurrentMaterial.PN;
                LotNumber: varSelectedLot.LotNumber;
                LotKey: varSelectedLot.LotKey;
                ActionType: "Delete";
                OldQty: varSelectedLot.Qty;
                NewQty: Blank();
                OldExpirationDate: varSelectedLot.ExpirationDate;
                NewExpirationDate: Blank();
                ChangedAt: Now();
                ChangedByName: User().FullName;
                ChangedByEmail: User().Email;
                Reason: txtDeleteReason.Text;
                Source: Coalesce(varSource; "Manual")
            }
        );
        Refresh(CM_Lots);
        Set(varSelectedLot; Blank());
        Reset(txtDeleteReason);
        UpdateContext({ ctxShowDelete: false });
        Notify("Lot deleted."; NotificationType.Success);
        Notify("Could not delete the lot."; NotificationType.Error)
    )
)

2) Byt btnSaveLot.OnSelect till exakt detta

With(
    {
        _pn: varCurrentMaterial.PN;
        _lot: Upper(Trim(txtLotNumber.Text));
        _qty: Value(txtQty.Text);
        _exp: dpExpirationDate.SelectedDate;
        _lotKey: varCurrentMaterial.PN & "|" & Upper(Trim(txtLotNumber.Text));
        _source: Coalesce(varSource; "Manual")
    };
    If(
        IsBlank(_lot) || IsBlank(_exp) || !IsNumeric(txtQty.Text) || _qty <= 0;
        Notify("Enter Lot Number, Expiration Date, and Qty greater than 0."; NotificationType.Error);

        If(
            ctxEditMode = "Add";

            If(
                !IsBlank(LookUp(CM_Lots; LotKey = _lotKey && IsActive = true));
                Notify("This Lot Number already exists for this PN."; NotificationType.Error);

                IfError(
                    Set(
                        varSelectedLot;
                        Patch(
                            CM_Lots;
                            Defaults(CM_Lots);
                            {
                                Title: _pn;
                                PN: _pn;
                                LotNumber: _lot;
                                LotKey: _lotKey;
                                ExpirationDate: _exp;
                                Qty: _qty;
                                IsActive: true;
                                ChangedAt: Now();
                                ChangedByName: User().FullName;
                                ChangedByEmail: User().Email
                            }
                        )
                    );
                    Patch(
                        CM_Transactions;
                        Defaults(CM_Transactions);
                        {
                            Title: _pn;
                            PN: _pn;
                            LotNumber: _lot;
                            LotKey: _lotKey;
                            ActionType: "Add";
                            OldQty: Blank();
                            NewQty: _qty;
                            OldExpirationDate: Blank();
                            NewExpirationDate: _exp;
                            ChangedAt: Now();
                            ChangedByName: User().FullName;
                            ChangedByEmail: User().Email;
                            Reason: Blank();
                            Source: _source
                        }
                    );
                    Refresh(CM_Lots);
                    Reset(txtLotNumber);
                    Reset(dpExpirationDate);
                    Reset(txtQty);
                    UpdateContext(
                        {
                            ctxShowEdit: false;
                            ctxEditMode: Blank()
                        }
                    );
                    Notify("Lot added."; NotificationType.Success);
                    Notify("Could not add the lot."; NotificationType.Error)
                )
            );

            If(
                _lotKey <> varSelectedLot.LotKey &&
                !IsBlank(LookUp(CM_Lots; LotKey = _lotKey && IsActive = true));
                Notify("This Lot Number already exists for this PN."; NotificationType.Error);

                IfError(
                    Set(
                        varSelectedLot;
                        Patch(
                            CM_Lots;
                            varSelectedLot;
                            {
                                Title: _pn;
                                PN: _pn;
                                LotNumber: _lot;
                                LotKey: _lotKey;
                                ExpirationDate: _exp;
                                Qty: _qty;
                                ChangedAt: Now();
                                ChangedByName: User().FullName;
                                ChangedByEmail: User().Email
                            }
                        )
                    );
                    Patch(
                        CM_Transactions;
                        Defaults(CM_Transactions);
                        {
                            Title: _pn;
                            PN: _pn;
                            LotNumber: _lot;
                            LotKey: _lotKey;
                            ActionType: "Change";
                            OldQty: varSelectedLot.Qty;
                            NewQty: _qty;
                            OldExpirationDate: varSelectedLot.ExpirationDate;
                            NewExpirationDate: _exp;
                            ChangedAt: Now();
                            ChangedByName: User().FullName;
                            ChangedByEmail: User().Email;
                            Reason: Blank();
                            Source: _source
                        }
                    );
                    Refresh(CM_Lots);
                    Reset(txtLotNumber);
                    Reset(dpExpirationDate);
                    Reset(txtQty);
                    UpdateContext(
                        {
                            ctxShowEdit: false;
                            ctxEditMode: Blank()
                        }
                    );
                    Notify("Lot updated."; NotificationType.Success);
                    Notify("Could not update the lot."; NotificationType.Error)
                )
            )
        )

⸻

3) Varför detta ska hjälpa

I din Studio verkar det vara så här:
	•	YAML / code view visar gärna kommatecken
	•	formula bar i din miljö vill ha semikolon

Så när du klistrar in kommateckens-versioner i formelfältet får du röda streck över hela record-blocket.

⸻

4) Om det fortfarande är rött efter detta

Då är det inte separatorerna längre. Då är det nästan säkert ett kolumnnamn i CM_Transactions.

Testa då dessa en och en i en tillfällig label:

First(CM_Transactions).OldExpirationDate

First(CM_Transactions).NewExpirationDate

First(CM_Transactions).Reason

First(CM_Transactions).Source

Den första som blir röd är den verkliga boven.
