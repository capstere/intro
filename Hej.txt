Best-practice beslut

Vi gör appen med 5 screens:
	1.	scrHome
	2.	scrSearchResults
	3.	scrLots
	4.	scrEditLot
	5.	scrDeleteLot

Och vi använder inte Back() i huvudflödet. Back() följer historiken, medan Navigate() ger ett kontrollerat flöde. Det är viktigare här, eftersom vi vill att användaren alltid ska hamna på rätt ställe efter Add/Change/Delete. Microsoft beskriver också att Back följer den faktiska navigeringshistoriken, medan Navigate skickar användaren till en bestämd screen och kan bära med sig state via context variables.  ￼

Vi använder också:
	•	Barcode reader på scrHome, eftersom den använder native scanner på iOS/Android/Windows och läser ut värdet via Barcodes/Value. Den stöds inte i vanlig webbläsare, så telefon är rätt målplattform.  ￼
	•	Patch för create/update/delete-logik, eftersom vi skriver till både CM_Lots och CM_Transactions.  ￼
	•	IfError runt submit/delete-blocket för robust felhantering.  ￼
	•	Reset för att nollställa inputs när man går in i eller lämnar edit/delete-flöden.  ￼

	•	validerar

If(
    IsBlank(Trim(txtLotNumber.Text)) || IsBlank(dpExpirationDate.SelectedDate) || !IsNumeric(txtQty.Text) || Value(txtQty.Text) <= 0,
    Notify("Enter Lot Number, Expiration Date, and Qty greater than 0.", NotificationType.Error),

    Set(varBusy, true);

    If(
        varMode = "Add",

        If(
            !IsBlank(
                LookUp(
                    CM_Lots,
                    LotKey = varCurrentMaterial.PN & "|" & Upper(Trim(txtLotNumber.Text)) && IsActive = true
                )
            ),
            Notify("This Lot Number already exists for this PN.", NotificationType.Error);
            Set(varBusy, false),

            IfError(

                    CM_Transactions,

                        PN: varCurrentMaterial.PN,

Del 7 — hur det här flödet beter sig “bakåt”

Det här är det du bad om särskilt.

Om användaren trycker Cancel
	•	från scrEditLot → tillbaka till scrLots
	•	från scrDeleteLot → tillbaka till scrLots

Om användaren sparar
	•	Add → tillbaka till scrLots, ny lot vald
	•	Change → tillbaka till scrLots, uppdaterad lot vald

Om användaren tar bort
	•	tillbaka till scrLots, lot-listan uppdaterad, ingen vald lot

Om användaren vill lämna materialet
	•	btnNewSearch → tillbaka till scrHome

Det här är ett stängt, logiskt flöde. Ingen “mystisk back-stack”.
