Ja. Här är en ren, mergad Sorterman för Ctrl+Shift+A som ska klara båda fallen:

1. Gammal/normal CSV: data är redan uppdelad i kolumner → makrot hoppar över Text to Columns och sorterar direkt.
2. Ny/problem-CSV: all data ligger ihop i kolumn A → makrot kör Text to Columns med Tab + Comma + General, sedan sorterar på kolumn C från rad 10.
3. Tab och comma blandat: eftersom både Tab:=True och Comma:=True används.

Det gamla originalmakrot sorterar på kolumn C från C10 och använder rad 10 som start på sorteringsområdet, så jag behåller samma huvudlogik men gör den säkrare och lägger till automatisk split före sorteringen.

Slutgiltig mergad Sorterman-kod

Option Explicit
Sub Sorterman_TextToColumnsIfNeeded()
'
' Delar upp kolumn A endast om det behövs.
'
' Gör inget om kolumn C redan har data från rad 10 och nedåt.
' Om kolumn C saknar data och kolumn A innehåller comma/tab,
' körs Text to Columns:
' Delimited -> Tab + Comma -> General -> Finish
'
    Dim lastRowA As Long
    Dim lastRowC As Long
    Dim checkToRow As Long
    Dim r As Long
    Dim cellText As String
    Dim needSplit As Boolean
    lastRowA = Cells(Rows.Count, "A").End(xlUp).Row
    lastRowC = Cells(Rows.Count, "C").End(xlUp).Row
    If lastRowA < 1 Then Exit Sub
    'Om kolumn C redan har data från rad 10 och nedåt,
    'antar vi att CSV redan är korrekt uppdelad.
    If lastRowC >= 10 Then
        If Application.WorksheetFunction.CountA(Range("C10:C" & lastRowC)) > 0 Then
            Exit Sub
        End If
    End If
    'Om kolumn C saknar data, kontrollera om kolumn A innehåller delimiter.
    checkToRow = lastRowA
    If checkToRow > 50 Then checkToRow = 50
    For r = 1 To checkToRow
        cellText = CStr(Cells(r, "A").Value)
        If InStr(1, cellText, ",", vbBinaryCompare) > 0 _
        Or InStr(1, cellText, vbTab, vbBinaryCompare) > 0 Then
            needSplit = True
            Exit For
        End If
    Next r
    If needSplit = False Then Exit Sub
    'Motsvarar manuellt:
    'Markera kolumn A
    'Data -> Text to Columns -> Delimited -> Tab + Comma -> General -> Finish
    Columns("A:A").Select
    Selection.TextToColumns _
        Destination:=Range("A1"), _
        DataType:=xlDelimited, _
        TextQualifier:=xlTextQualifierDoubleQuote, _
        ConsecutiveDelimiter:=False, _
        Tab:=True, _
        Semicolon:=False, _
        Comma:=True, _
        Space:=False, _
        Other:=False, _
        FieldInfo:=Array(1, 1), _
        TrailingMinusNumbers:=True
End Sub
Sub Sorterman()
'
' Sorterman Macro
'
' Keyboard Shortcut: Ctrl+Shift+A
'
' Mergad version:
' - Gammal CSV, redan uppdelad: sorterar direkt på kolumn C
' - Ny CSV, allt i kolumn A: Text to Columns med Tab + Comma, sedan sortering på kolumn C
'
    Dim lastRowA As Long
    Dim lastRowC As Long
    Dim lastRow As Long
    Dim lastCol As Long
    On Error GoTo ErrorHandler
    Application.ScreenUpdating = False
    'Steg 1:
    'Dela upp kolumn A endast om kolumn C saknar data och A innehåller comma/tab.
    Sorterman_TextToColumnsIfNeeded
    'Steg 2:
    'Hitta sista rad efter eventuell split.
    lastRowA = Cells(Rows.Count, "A").End(xlUp).Row
    lastRowC = Cells(Rows.Count, "C").End(xlUp).Row
    lastRow = lastRowA
    If lastRowC > lastRow Then lastRow = lastRowC
    If lastRow < 10 Then
        MsgBox "Sortering avbruten. Ingen data hittades från rad 10 och nedåt.", vbExclamation, "Sorterman"
        GoTo CleanExit
    End If
    'Efter eventuell split måste kolumn C innehålla något att sortera på.
    If Application.WorksheetFunction.CountA(Range("C10:C" & lastRow)) = 0 Then
        MsgBox "Sortering avbruten. Kolumn C saknar data även efter eventuell Text to Columns." & vbCrLf & _
               "Kontrollera CSV-filen manuellt.", vbExclamation, "Sorterman"
        GoTo CleanExit
    End If
    'Sista använda kolumn på rad 10.
    lastCol = Cells(10, Columns.Count).End(xlToLeft).Column
    If lastCol < 3 Then
        MsgBox "Sortering avbruten. Filen verkar inte ha minst tre kolumner.", vbExclamation, "Sorterman"
        GoTo CleanExit
    End If
    'Steg 3:
    'Samma syfte som gamla Sorterman: autofit och sortering på kolumn C från rad 10.
    Cells.Select
    Cells.EntireColumn.AutoFit
    ActiveSheet.Sort.SortFields.Clear
    ActiveSheet.Sort.SortFields.Add _
        Key:=Range("C10:C" & lastRow), _
        SortOn:=xlSortOnValues, _
        Order:=xlAscending, _
        DataOption:=xlSortNormal
    With ActiveSheet.Sort
        .SetRange Range(Cells(10, 1), Cells(lastRow, lastCol))
        .Header = xlGuess
        .MatchCase = False
        .Orientation = xlTopToBottom
        .SortMethod = xlPinYin
        .Apply
    End With
    ActiveSheet.Range("A10").Select
CleanExit:
    Application.ScreenUpdating = True
    Exit Sub
ErrorHandler:
    Application.ScreenUpdating = True
    MsgBox "Sorterman misslyckades: " & Err.Description, vbExclamation, "Sorterman"
End Sub

Hur den avgör vad den ska göra

Filtyp	Vad makrot ser	Vad det gör
Gammal CSV, redan uppdelad	Kolumn C har data från rad 10	Hoppar över Text to Columns och sorterar
Ny CSV, allt i kolumn A	Kolumn C saknar data, kolumn A har comma/tab	Kör Text to Columns och sorterar
Konstig CSV utan kolumn C efter split	Kolumn C fortfarande tom	Stoppar med felmeddelande
Blandad delimiter i kolumn A	Tab och comma finns	Split med båda
