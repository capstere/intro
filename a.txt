Fel 4: D Lagging måste felsökas mot Production Orders, inte mot Delivery Daily

Här ska vi vara väldigt tydliga:
	•	QDIP_Delivery_Daily hjälper dig med:
	•	D_Leading_LotsWIP_10hPlus
	•	delivery causes
	•	D Lagging kommer från:
	•	Cepheid | Production orders
	•	Lead-time_Num
	•	Delivery_Date

Så om D Lagging “inte stämmer” måste vi kontrollera:
	•	Delivery_Date är korrekt calculated column
	•	Lead-time_Num är korrekt decimal
	•	relationen Cepheid | Production orders[Delivery_Date] -> Dim_Date[Date] är aktiv
	•	measuret verkligen använder Lead-time_Num >= 20

Jag skulle låta dessa stå kvar så här:

D_Lagging_TotalLots =
CALCULATE(
    COUNTROWS('Cepheid | Production orders'),
    NOT(ISBLANK('Cepheid | Production orders'[ROBAL - Actual end date/ time])),
    NOT(ISBLANK('Cepheid | Production orders'[Lead-time_Num]))
)

D_Lagging_LotsOver20h =
CALCULATE(
    COUNTROWS('Cepheid | Production orders'),
    NOT(ISBLANK('Cepheid | Production orders'[ROBAL - Actual end date/ time])),
    NOT(ISBLANK('Cepheid | Production orders'[Lead-time_Num])),
    'Cepheid | Production orders'[Lead-time_Num] >= 20
)

D_Lagging_DayStatus =
VAR _total = [D_Lagging_TotalLots]
VAR _over = [D_Lagging_LotsOver20h]
RETURN
    IF(
        ISBLANK(_total),
        BLANK(),
        IF(_over = 0, "Green", "Red")
    )
